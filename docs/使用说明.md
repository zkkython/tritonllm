# 使用说明（中文版）

本文档给最终用户直接复制粘贴使用。

## 0. 准备环境（首次执行）

```bash
cd /Users/kason/pythonwork/tritonllm
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
playwright install chromium
```

## 1. 普通抓取（推荐先试）

先设置目标链接（以后只改这一行）：

```bash
URL="https://example.com/your-article"
```

执行导出：

```bash
./.venv/bin/python scripts/medium_to_markdown.py \
  "$URL" \
  --out-dir exports \
  --export-html \
  --code-mode html
```

输出结果：
- Markdown：`exports/<name>.md`
- 高保真 HTML：`exports/<name>.html`
- 图片目录：`exports/<name>_assets/`

## 2. 指定输出文件名（避免文件名变化）

```bash
URL="https://example.com/your-article"

./.venv/bin/python scripts/medium_to_markdown.py \
  "$URL" \
  --out-dir exports \
  --filename article-001 \
  --export-html \
  --code-mode html
```

## 3. 遇到登录/安全验证页（先人工过一次）

```bash
URL="https://example.com/your-article"

./.venv/bin/python scripts/medium_to_markdown.py \
  "$URL" \
  --out-dir exports \
  --show-browser \
  --manual-verify-timeout-ms 180000 \
  --storage-state-out ./medium_state.json \
  --export-html \
  --code-mode html
```

说明：
- 浏览器弹出后，手动完成验证/登录。
- 完成后脚本会继续执行并保存会话到 `medium_state.json`。

## 4. 复用会话自动抓取

```bash
URL="https://example.com/your-article"

./.venv/bin/python scripts/medium_to_markdown.py \
  "$URL" \
  --out-dir exports \
  --storage-state-in ./medium_state.json \
  --idle-timeout-ms 3000 \
  --retries 5 \
  --export-html \
  --code-mode html
```

## 5. 验证反复过期（最稳方案：连接你正在用的 Chrome）

### 5.1 启动可调试 Chrome

```bash
/Applications/Google\ Chrome.app/Contents/MacOS/Google\ Chrome \
  --remote-debugging-port=9222 \
  --user-data-dir=/tmp/chrome-medium
```

### 5.2 在该 Chrome 中手动打开文章并完成验证
- 确保正文已经可见。

### 5.3 脚本连接该浏览器并抓当前标签页

```bash
URL="https://example.com/your-article"

./.venv/bin/python scripts/medium_to_markdown.py \
  "$URL" \
  --out-dir exports \
  --cdp-url http://127.0.0.1:9222 \
  --use-existing-page \
  --export-html \
  --code-mode html
```

## 6. 常用参数速查

- `--filename article-001`：指定输出文件名
- `--retries 5`：失败重试次数
- `--timeout-ms 120000`：页面导航超时
- `--idle-timeout-ms 3000`：网络空闲等待时长（超时会继续）
- `--cookie-file ./cookies.json`：注入 cookies 文件
- `--cookie uid=xxx --cookie sid=yyy`：命令行注入 cookie
- `--show-browser`：显示浏览器窗口
- `--storage-state-in/out`：会话复用
- `--cdp-url + --use-existing-page`：连接现有浏览器抓当前页
- `--export-html`：额外导出高保真 HTML
- `--code-mode html`：代码块用 HTML 保真（复杂页面更稳）

## 7. 推荐实践

1. 阅读优先：看 `exports/<name>.html`。
2. 检索/RAG 优先：用 `exports/<name>.md`。
3. 如果代码块排版异常，优先使用 `--code-mode html`。
4. 如果目标站点风控严格，优先使用 CDP 方案（第 5 节）。

## 8. 常见报错处理

### 报错：Detected Cloudflare/security verification page
- 先按第 3 节人工验证并保存 `storage_state`。
- 仍失败就用第 5 节 CDP 方案。

### 报错：ModuleNotFoundError
- 使用 `./.venv/bin/python ...` 执行，或先 `source .venv/bin/activate`。

### 报错：图片路径不显示
- 文档内请使用不含 `..` 的相对路径或站点允许的绝对路径。
- 本项目建议图片与文档同级目录组织（例如 `docs/image/...`）。
